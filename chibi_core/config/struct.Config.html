Expand description

Global config from ~/.chibi/config.toml Note: This is the core config. Presentation fields (image, markdown_style, render_markdown) are handled by the CLI layer. All fields are optional with sensible defaults — config.toml itself is optional.

§auto_compact: bool§auto_compact_threshold: f32§reflection_enabled: bool§reflection_character_limit: usize

Maximum characters for reflection tool output

§fuel: usize

Total fuel budget for the agentic loop (tool rounds, continuations, empty responses). Set to 0 to disable fuel tracking entirely (unlimited mode).

§fuel_empty_response_cost: usize

Fuel cost of an empty response (high cost prevents infinite empty loops). Ignored when fuel = 0 (unlimited mode).

§username: String§lock_heartbeat_seconds: u64

Lock heartbeat interval in seconds. Intentionally global-only (not in ResolvedConfig) since lock behaviour must be consistent regardless of active context.

§rolling_compact_drop_percentage: f32§tool_output_cache_threshold: usize

Threshold (in chars) above which tool output is cached

§tool_cache_max_age_days: u64

Maximum age in days for cached tool outputs

§auto_cleanup_cache: bool

Automatically cleanup old cache entries on exit

§tool_cache_preview_chars: usize

Number of preview characters to show in truncated message

§file_tools_allowed_paths: Vec<String>

Paths allowed for file tools (empty = defaults to cwd at runtime)

§api: ApiParams

API parameters (temperature, max_tokens, etc.)

§storage: StorageConfig

Storage configuration for partitioned context storage

§fallback_tool: String

Fallback tool when LLM doesn’t call call_agent/call_user explicitly

§tools: ToolsConfig

Global tool filtering configuration (include/exclude/exclude_categories)

§vfs: VfsConfig

Global VFS configuration. Not in ResolvedConfig — the VFS backend is a startup-time singleton and not per-context configurable.

§url_policy: Option<UrlPolicy>

URL security policy for sensitive URL handling

§subagent_cost_tier: String

Cost tier used when resolving subagent presets (e.g. “free”, “standard”, “premium”). Controls which tier of ratatoskr presets spawn_agent resolves against.
