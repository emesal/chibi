The current design (approach A) has Vfs call the backend directly. A future evolution (approach C) would wrap backends in composable middleware layers (logging, caching, etc.) a la tower. The trait signature is designed to be compatible with that transition.

Methods return Pin<Box<dyn Future>> instead of impl Future so that Box<dyn VfsBackend> works. This enables multi-backend mounting where Vfs selects a backend at runtime via longest-prefix match.
