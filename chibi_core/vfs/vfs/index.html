Vfs: the permission-enforcing router.

This is the single entry point for all VFS operations. It validates permissions based on caller identity and path zone, then delegates to the appropriate backend via longest-prefix mount matching.

§Multi-backend mounting

Use Vfs::builder(site_id) to mount different backends at different path prefixes. The backend with the longest matching prefix handles the operation; the mount prefix is stripped from the path before delegation.

Vfs::new(backend, site_id) mounts a single backend at / (convenience for the common single-backend case).

§Middleware layers

Future evolution: composable tower-style layers wrapping the backend (approach C). The public API on Vfs stays unchanged.
