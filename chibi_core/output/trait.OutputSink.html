Expand description

Abstraction over how command results and diagnostics are presented.

chibi-cli implements this with OutputHandler (text to stdout/stderr, interactive TTY). chibi-json implements this with JsonOutputSink (JSONL to stdout/stderr, auto-approve).

fn emit_entry(&self, entry: &TranscriptEntry) -> Result<()>

Emit a transcript entry for display.

Each sink formats entries appropriately for its output medium: CLI renders human-readable text, JSON emits structured JSONL.

Source

fn confirm(&self, prompt: &str) -> bool

Prompt the user for confirmation. Returns true if confirmed. Programmatic sinks (e.g. chibi-json) should auto-approve (return true).

Provided Methods§

Source

fn emit_markdown(&self, content: &str) -> Result<()>

Emit content that may contain markdown.

CLI renders this via streamdown; JSON emits raw text. The default implementation falls back to emit_result().

Source

fn emit_done(&self, result: &Result<()>)

Signal command completion. Called once, after all output has been emitted.

Default: no-op — chibi-cli handles completion via its own UX.
